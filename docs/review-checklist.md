# Review Checklist

## Purpose

Use this checklist when reviewing a specific proofreading output.

This is the practical per-run review pass that supports the manual validator decision.

---

## Run verification

Before judging output, confirm:

- the correct prompt file was used
- the correct input file was used
- the output file saved completely
- the output file is non-empty
- the run metadata matches the intended run when registry logging applies

---

## Output contract

Confirm:

- output returned in the requested format
- required output sections are present
- no added commentary appears outside the expected format
- no visible reasoning appears in the output

---

## Protected terms and canon safety

Confirm:

- all character names preserved
- all place names preserved
- all faction names preserved
- all title or honorific terms preserved
- all special canon terms preserved
- no obvious canon drift introduced

---

## Proofreading behavior

Check whether:

- grammar improved where needed
- spelling improved where needed
- punctuation improved where needed
- obvious usage errors were corrected
- no unnecessary rewrites were introduced
- editing restraint was appropriate for a proofread task

---

## Literary integrity

Confirm:

- tone preserved
- voice preserved
- rhythm mostly preserved
- meaning preserved
- imagery not degraded

---

## Minor drift check

Note whether any of the following occurred without breaking acceptability:

- small phrasing differences
- light sentence smoothing
- minor wording drift
- small punctuation cleanup differences

If present, decide whether they belong under `accept with notes`.

---

## Final judgment

Use one of:

- `accept`
- `accept with notes`
- `reject`

Always record why, especially for `accept with notes` and `reject`.
